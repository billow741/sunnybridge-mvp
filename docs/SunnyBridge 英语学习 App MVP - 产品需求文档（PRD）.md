1. 产品背景

SunnyBridge 是一家面向少儿的英语培训机构，目前使用独立的 CRM 系统管理课程排期、教师课时、学生信息等教务工作。随着业务增长，家长和学生需要一个集中的移动端入口来查看课程安排、获取课后反馈、进行课后阅读和学习。

本 PRD 描述的是全新的独立 App MVP，与现有 CRM 无技术耦合。

1.1 现有痛点

**课程信息分散**
• 描述: 家长通过微信群、短信、电话等多种渠道接收课程信息
• 影响: 容易遗漏、效率低

**反馈查看不便**
• 描述: 课后反馈以文字形式发送给家长，无法集中查看历史
• 影响: 难以追踪学习进度

**阅读资源无体系**
• 描述: 英语阅读材料分散在各类群文件或外部平台
• 影响: 孩子缺乏系统的阅读培养

**缺乏学习闭环**
• 描述: 上课、反馈、复习三个环节各自为政
• 影响: 学习效果难以量化

1.2 项目信息

**产品名称**
• 内容: SunnyBridge 英语学习 App

**目标平台**
• 内容: iOS、Android（Flutter 跨平台）

**上线时间**
• 内容: 2 个月（8 周）

**技术栈**
• 内容: Flutter + FastAPI + Supabase

2. 问题定义

2.1 核心问题陈述

 如何让家长高效参与孩子的英语学习过程，让孩子形成"上课 → 反馈 → 阅读/复习"的稳定学习习惯？

2.2 用户痛点拆解

**家长**
• 核心痛点: 无法及时得知课程变动，反馈信息零散
• 当前做法: 微信群通知
• 期望改进: 统一 App 内查看所有课程和反馈

**学生**
• 核心痛点: 课后不知道读什么、怎么复习
• 当前做法: 家长自行寻找材料
• 期望改进: App 内系统化的阅读馆和资源库

**教师**
• 核心痛点: 课后需要单独发送反馈给家长，容易遗漏
• 当前做法: 微信/短信发送
• 期望改进: 在 App 内统一提交，家长实时查看

**教务**
• 核心痛点: 课程安排需要人工通知多方
• 当前做法: 手动群通知
• 期望改进: App 内自动同步

3. 产品目标

3.1 核心目标（Must）

G1
• 目标描述: 建立"上课 → 反馈 → 阅读/复习"的学习闭环
• 衡量指标: 课程完成率 → 反馈查看率 → 阅读馆使用率

G4
• 目标描述: 为后续留存、续费、数据分析打基础

• 目标: 优化阅读体验，增加阅读时长统计

V1.2
• 目标: 增加学习积分/成就系统

V2.0
• 目标: AI 口语评分、互动练习

5. 核心使用场景

场景 1：家长查看今日课程

家长打开 App → 切换至"家长模式" → 看到今日课程卡片 → 
点击课程 → 查看详情（时间、教师、腾讯会议链接）→ 
课前 15 分钟收到推送提醒 → 点击"进入教室"跳转到腾讯会议


场景 2：教师提交课后反馈

教师打开教师端 App → 查看今日课程列表 → 找到已完成的课程 → 
点击"填写反馈" → 输入课堂内容、作业、备注 → 
选择课程状态"已完成" → 提交 → 家长端实时收到通知


场景 3：学生课后阅读

学生切换至"学生模式" → 进入"英语阅读馆" → 
按级别浏览（L1-L6）或按分类浏览 → 选择一本绘本 → 
翻页阅读、高亮生词、做笔记 → 阅读完成，下次自动续读


场景 4：教务安排新课程

教务登录后台 → 课程管理 → 新建课程 → 
选择教师、学生、日期、时间 → 输入腾讯会议链接 → 
保存 → 家长和教师端自动同步显示新课程

9.1 模块 M1：登录与绑定

9.1.1 功能目标

**目标**
• 内容: 让家长通过手机号快速注册登录，并完成孩子信息绑定

**优先级**
• 内容: P0（阻塞性）

**适用角色**
• 内容: 家长

9.1.2 前置条件

用户已下载安装 App
• 说明: iOS / Android

用户拥有可接收短信的手机号
• 说明: 中国大陆手机号

后端短信服务已配置
• 说明: 接入短信平台

9.1.3 核心业务规则

R1
• 规则描述: 手机号格式验证：1[3-9]\d{9}

R2
• 规则描述: 验证码有效期 5 分钟，错误 3 次后需重新发送

R3
• 规则描述: 新用户需完成孩子信息填写方可进入首页

R4
• 规则描述: 一个手机号对应一个家长账号，绑定一个孩子

R5
• 规则描述: JWT Token 有效期 7 天，过期后自动刷新

9.1.4 页面元素

登录页

手机号输入框
• 类型: Input
• 说明: 数字键盘，限制 11 位

验证码发送按钮
• 类型: Button
• 说明: 点击后 60 秒倒计时

验证码输入框
• 类型: Input
• 说明: 6 位数字

登录按钮
• 类型: Button
• 说明: 手机号+验证码验证通过后可用

注册页（绑定孩子）

孩子姓名
• 类型: Input
• 说明: 必填

所在年级
• 类型: Select
• 说明: 幼儿园/一年级~六年级

孩子头像
• 类型: Image Picker
• 说明: 可选，默认头像

完成按钮
• 类型: Button
• 说明: 信息填写完整后可用

1

• 处理方式: Toast 提示"网络连接失败，请检查网络"

AC3

AC4
